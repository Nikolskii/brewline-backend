## Назначение репозитория
Backend Brewline: **REST API + real-time поток статусов заказов**. Центральный сервис — держит цифровую
очередь заказов и рассылает изменения статуса во все точки отображения (клиент, бариста, экран).

**⭐ Ключевая роль в архитектуре — источник истины контракта.** OpenAPI-спека живёт здесь; фронты
(`brewline-display` и др.) **генерируют** из неё TypeScript-типы. Это значит: меняешь контракт — думаешь
о потребителях. (ADR 0008.)

## Стек
- **Node.js + Express + TypeScript** (TS strict).
- **MongoDB** — хранилище заказов (официальный драйвер `mongodb`). ADR 0002 (Mongo под модель заказов — открытый вопрос, помечать сомнения).
- **Real-time — SSE (Server-Sent Events)**, не WebSocket. Модель: «снапшот» — сервер шлёт актуальное
  состояние очереди, payload переиспользует доменный тип `Order` из той же OpenAPI (отдельного языка событий нет). ADR 0001.

## Границы инкремента 1 (walking skeleton)
Скоуп именно этого среза (декомпозиция — в vault, `planning/Декомпозиция — инкремент 1 …`):
- Модель `Order` в MongoDB + **автомат статусов** (I3).
- **OpenAPI-спека** REST + доменные типы (I4) — делаем в первую очередь, разблокирует фронт.
- `GET /orders` — снапшот очереди.
- Смена статуса заказа (эндпоинт).
- `GET /orders/stream` — SSE-поток снапшотов.
Вне скоупа среза 1: создание заказов из кассы/веб-формы, аутентификация, деплой (это срез 2).

## Как связан с соседями
- `brewline-display` потребляет: `GET /orders` (снапшот) + `GET /orders/stream` (SSE) и генерит типы из нашей OpenAPI.
- `brewline-infra` даёт MongoDB локально через `docker-compose` — backend к ней подключается по `MONGO_URL` из env.

## Локальный запуск (наполнится по мере кода)
- Mongo поднимается из `brewline-infra` (`docker compose up`).
- Конфиг — через переменные окружения (`MONGO_URL`, `PORT`). Реальные значения — в `.env` (не коммитить), пример — в `.env.example`.

## Безопасность (репо публичный)
- **Никогда не коммитить секреты**: `.env`, `MONGO_URL` с паролем, токены. Только плейсхолдеры и `.env.example`.

## Конвенции
- TypeScript strict; явные доменные типы.
- ADR пишем, когда упёрлись в развилку (скилл `adr`), а не заранее.
